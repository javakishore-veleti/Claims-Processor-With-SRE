Docker Optional Services (Env Vars) Resource Control
Control which Docker containers start via environment variables. Default stack runs Postgres, Kafka, Prometheus, Alertmanager, Grafana, Jaeger, Zipkin. Optional services are disabled by default to save laptop resources.
Environment Variable Default Service RAM Saved
CLAIMS_PROCESSOR_APP_LOCAL_KAFKAfalseKafka + Zookeeper (event streaming)~450MB
CLAIMS_PROCESSOR_APP_LOCAL_REDISfalseRedis (cache)~10MB
CLAIMS_PROCESSOR_APP_LOCAL_ELASTICSEARCHfalseElasticsearch (search + logs)~1GB
CLAIMS_PROCESSOR_APP_LOCAL_KIBANAfalseKibana (log analytics UI)~540MB
CLAIMS_PROCESSOR_APP_LOCAL_FILEBEATfalseFilebeat (log shipper)~55MB
CLAIMS_PROCESSOR_APP_LOCAL_WIREMOCKfalseWireMock (API mocking)~85MB
CLAIMS_PROCESSOR_APP_LOCAL_OLLAMAfalseOllama (local AI inference)~11MB
Usage Examples
npm run devops:start:all                                    # Default (no optional services)
CLAIMS_PROCESSOR_APP_LOCAL_REDIS=true npm run devops:start:all    # Enable Redis
CLAIMS_PROCESSOR_APP_LOCAL_ELASTICSEARCH=true CLAIMS_PROCESSOR_APP_LOCAL_KIBANA=true npm run devops:start:all  # Enable ELK
Fallbacks when disabled: Redis → Caffeine in-memory cache | Elasticsearch → JPA database queries | Kibana/Filebeat → Console JSON logs | WireMock → Direct API calls | Ollama → Mock AI extraction
